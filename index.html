TECHNICAL CONTENT SAMPLES

I believe writing successful technical content depends on understanding three things: the concept (what am I writing about?), the audience (who am I writing for?), and the goal (what do I want this content to accomplish?). Here are some samples of content I’ve produced, explored through the lens of this simple framework.

Forward customer data to Snowflake for analysis and storage

A data forwarding tutorial written for mParticle.

CONCEPT: This tutorial focuses on an mParticle integration with Snowflake that allows customers to forward cleansed and unified customer data from their CDP to their Data Warehouse. This integration relieves data engineers of the need to build a custom data connection between these two systems, and can be implemented by a simple UI-based workflow.

AUDIENCE: This content is primarily intended for data engineers at mParticle’s customer companies who are responsible for integrating the various systems in their data architecture.

GOALS: The primary goal of this piece was to encourage engineers at mParticle’s customer companies to take advantage of this integration, and allow them to use this tutorial alone to implement it (hence the inclusion of SQL queries that perform the necessary provisioning on the Snowflake side). Additionally, the tutorial also aimed to convey the benefits of establishing this data connection through the use case and examples it highlights.

How to install and start using FFmpeg in under 10 minutes

An installation guide written for Editframe, an API-based video editing tool that allows developers to manipulate and embed video files using JavaScript libraries.

CONCEPT: The Editframe content team wanted to publish an authoritative blog post detailing how to install and begin using FFmpeg––a popular tool that enables many of the same video editing use cases as Editframe.

AUDIENCE: This post is geared towards anyone––both developers and semi-technical hobbyists––who wants to quickly install and begin using FFmpeg.

GOALS: The primary goal of this piece is to enable the reader to easily install FFmpeg on their platform of choice and acquire basic knowledge of FFmpeg’s commands and functionality.
